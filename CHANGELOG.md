## [0.1.5] - 2026-04-21

### Fixed

- **채팅 입력 포커스 해제(`content.js`)**: 치지직 채팅/댓글 입력창에 포커스된 상태에서 `Esc`를 누르면 입력 포커스를 해제(`blur`)해, 곧바로 메모 단축키를 눌러 메모 입력창을 열 수 있게 했습니다.
- **VOD 마커 호버·하단바 홀드(`content.js`)**: 마커 툴팁이 `pointer-events: none`이고 마커 레이어가 얇아 실제 타깃이 진행 바로 잡히는 경우 `cmmPointerOverMarkerUi`가 꺼지던 문제를, 마커·툴팁 구역 기하 검사·`pointerout` 시 좌표 재검사·`syncOverlayVisibility`의 `markerUiGeometryHold`로 보완했습니다.
- **VOD 마커 위에서 PZP 하단 크롬 유지(nudge, `content.js`)**: `elementFromPoint`가 마커만 가리키던 문제는 `elementsFromPoint` 스택에서 확장 UI를 건너 뛴 뒤 PZP 노드를 고르게 했고, 진행 슬라이더 서브트리(`isPzpVodProgressSeekNudgeUnsafe`)에는 합성 이벤트를 보내지 않아 호버 미리보기가 뜨지 않게 했습니다. 슬라이더를 제외한 뒤 타이머가 잘 안 깨지는 경우를 위해 하단 그림자·VOD 시간·볼륨 등 안전한 크롬 조각에 `clientPointSnappedInsideEl`로 좌표를 스냅해 디스패치합니다.
- **라이브 첫 기록 시각이 `23시간`으로 튀는 현상(`content.js`)**: `getTimelineSecond()`에서 JSON/캐시 기반 wall 경과와 DOM/PZP 경과가 크게 벌어지는 비정상 패턴(예: wall이 수시간 이상 크고 DOM은 방송 초반)일 때, 현재 라이브 시작시각 캐시를 즉시 무효화하고 DOM/PZP 경과를 우선 사용하도록 보강했습니다. 이로써 새 방송 시작 직후 이전 방송 캐시가 섞여 첫 메모 시간이 하루 단위로 크게 표시되는 문제를 줄였습니다.
- **라이브 시작시각 레거시 캐시 정리(`content.js`)**: `liveId` 단일 키(구버전 호환용) 읽기/쓰기를 중단하고, 확장 부팅 시 해당 레거시 키를 자동 정리하도록 `cleanupLegacyLiveStartCacheKeys()`를 추가했습니다. 이제 `live:${liveId}:${episodeKey}` 스코프 캐시만 사용해, 방송 갱신 후 전날 시작시각이 재사용되는 문제를 줄였습니다.
- **라이브 세션 엄격 매칭(`content.js`, `popup.js`)**: `liveStartMs`를 읽은 경우에는 시작시각 근접(2시간 퍼지) 세션만 같은 방송 후보로 인정하고, 최근 갱신(`updatedAt`)만으로 다른 방송 세션에 합쳐지는 경로를 제거했습니다. `liveStartMs`를 읽지 못한 경우에는 같은 `liveId`의 최신 세션을 사용하도록 정리했으며, 팝업의 라이브 세션 자동 선택 로직도 동일 기준으로 맞췄습니다.

## [0.1.4] - 2026-04-13

### Changed

- **라이브 시작 시각 캐시 구조 개선(`content.js`)**: `liveId` 단일 키 대신 `live:${liveId}:${episodeKey}` 스코프 키를 함께 사용해, 같은 채널에서 이전 방송 시작 시각 캐시가 다음 방송으로 섞이는 가능성을 낮췄습니다. 기존 저장값과의 호환을 위해 레거시 키(`liveId`)도 유지합니다.

### Fixed

- **라이브 메모 시간 초기 오프셋 이상**: `getLiveStartMs()`에서 캐시 값을 즉시 신뢰하지 않고 페이지 JSON(`openDate` 계열)과 즉시 교차검증하도록 보강해, 캐시와 실측 시작 시각 차이가 큰 경우 자동으로 갱신합니다.
- **라이브 캐시 무효화 범위 보강**: 특정 라이브 무효화 시 `liveId` 레거시 키뿐 아니라 해당 라이브의 스코프 키(`live:${liveId}:...`)까지 함께 제거해 잘못된 캐시 재사용을 줄였습니다.

## [0.1.3] - 2026-04-12

### Added

- 팝업 **별도 창으로 열기**: `windows` 권한으로 `popup.html?dock=1`을 작은 전용 창에 띄워, 툴바 팝업처럼 바깥 클릭으로 닫히지 않고 **창을 닫을 때까지** 메모 UI를 유지할 수 있습니다.
- 팝업 **세션 이름 수정**용 텍스트 입력 모달(`cmm-prompt-modal`): 브라우저 `prompt` 대신 확장 팝업 안에서 제목을 편집합니다.
- **수동 세션** 생성 시, 활성 치지직 탭 제목(`chzzkActivePageContext`, 라이브/VOD)을 우선해 `streamerName`·표시 `title`을 채우고, 없으면 스트리머 드롭다운 그룹을 사용합니다.

### Changed

- **팝업 세션 영역**: 스트리머 선택과 같이 세션 `<select>`를 전체 너비로 두고, **이름 수정·새 세션·세션 삭제** 버튼을 선택란 **아래 한 줄**(가로 배치)로 옮겼습니다.
- 확인 모달과 입력 모달의 **겹침 순서**를 `z-index`로 정리했습니다(확인이 입력 위에 표시).
- **`content.js` 라이브 세션**: `ensureCurrentSession`에 **퍼지 매칭**을 적용해, 같은 `sourceId`에 대해 `liveStartMs`가 2시간 이내로 어긋나거나 최근 2시간 이내에 갱신된 라이브 세션을 동일 방송으로 이어 씁니다. 일치가 없을 때만 `live:{채널}:{liveStartMs}` 형태로 새 세션을 만듭니다.
- **`content.js` 라이브 자동 기록**: `categoryWatchTimer` / `startCategoryWatch` / `stopCategoryWatch`를 **`metadataWatchTimer` / `startMetadataWatch` / `stopMetadataWatch`**로 바꾸고, 진입 시 `ensureLiveMetadataOnJoin`으로 통일했습니다. 5초 주기로 **카테고리·방제(`document.title`)·타임라인**을 함께 읽어 `checkAndInsertMetadataIfChanged`에서 한 번에 처리합니다(방제만·카테고리만·동시 변경 시 **한 줄**만 추가해 중복 기록을 줄임). 방제가 바뀌면 저장소의 `session.title`을 먼저 갱신합니다.
- **자동 기록 문구·형식**: 방제 비교·기록에는 탭 제목의 **방송 제목 부분만** 쓰고(스트리머명 제외), 방제 전용·카테고리+방제 동시 변경은 **`autoMeta`**(`"title"` / `"both"`)와 본문(방제 문자열만 또는 **`카테고리명/방제`** 한 줄)로 구분합니다. 신규 저장에는 `[방제 변경]` 등 **접두어를 붙이지 않습니다**(구형 접두어 데이터는 팝업 표시·TXT 보내기에서만 제거해 보여 줌).
- **팝업**: 열 때마다 활성 탭 기준으로 **시청 중 라이브·VOD에 맞는 세션**을 고르고, 라이브는 저장소 세션과 **`liveStartMs` 2시간 퍼지**로 맞춥니다. **새 세션** 기본값은 라이브/VOD 탭 제목에서 스트리머·표시 제목을 가져옵니다.
- **문서·팝업 UI**: 기능 설명을 **「라이브 카테고리 / 방송 제목 자동 감지」** 로 통일했습니다.
- **팝업 기록 목록**: `[방제 변경]`·`[카테고리/방제 변경]` 자동 줄의 유형 배지를 **「카테고리 변경」** 대신 **「방제 변경」**, **「카테고리/방제 변경」** 으로 표시합니다. VOD 마커 툴팁(`content.js`)도 같은 문구로 맞춥니다.
- **별도 창(`?dock=1`)**: 스트리머·세션·기록 목록(정렬 포함)만 보이는 단축 레이아웃, 세로 여백을 기록 목록에 할당, **`chrome.windows`로 가로 너비 유지** 및 `viewport`·`overflow-x`로 가로 스크롤·핀치 확대 완화.
- **`content.js` 정리**: 호출되지 않던 **`isNativePzpBottomControlsVisible`**, **`upsertSession`** 함수를 제거했습니다.

### Fixed

- **라이브 자동 감지**: `카테고리/방제 변경` 한 줄을 남긴 뒤 다음 주기에서, 여전히 **이전 일반 카테고리**와만 비교해 **`카테고리 변경`이 한 번 더 쌓이던 현상**을 고쳤습니다(기록을 뒤에서 읽을 때 복합 줄의 카테고리 구간을 반영합니다).
- **별도 창(`?dock=1`)**: 창 세로 크기에 맞춰 **기록 목록**이 남는 공간을 채우고, 스크롤은 목록 안에서만 생기도록 레이아웃을 조정했습니다.
- **라이브 타임라인(레이아웃 전환)**: 넓은 화면·전체 화면에서 일반 화면으로 바꾼 직후, 상단 경과 시각 DOM이 늦게 갱신될 때 **낡은 시각**으로 카테고리가 찍히던 경우를 줄이기 위해, `video_information`·`live_information_player`·PZP 등에서 읽은 값과 **시작 시각 기준 벽시계 경과** 중 **가장 큰(진행이 더 많이 된) 초**를 쓰도록 바꿨습니다.
- **라이브 방제·카테고리 동시 변경**: 한 주기에 카테고리와 방제가 함께 바뀔 때 **한 줄**만 남기도록 통합했습니다(`autoMeta: "both"` 등; 이전처럼 같은 시각에 두 줄이 생기지 않도록).
- **라이브 카테고리(넓은 화면·오탐)**: 플레이어 정보 바 **일반 텍스트·`#__NEXT_DATA__` 스캔**이 시청자 수 등을 잡을 수 있어 제거하고, `live_information_player_*` 안 **카테고리·게임 카탈로그 `a[href]`** 및 **인라인 스크립트 JSON**(`liveCategoryValue` 등)만 보강합니다.
- **VOD 타임라인 마커(전체/넓은 화면)**: 브라우저 전체화면 top layer 안에서 오버레이가 보이도록 마커/클러스터 팝 오버레이의 마운트 부모를 조정하고(`fullscreenchange` / `webkitfullscreenchange`), 일반·넓은 화면에서도 PZP 하단 UI에 가리지 않도록 마커 레이어 `z-index`를 상향했습니다.
- **VOD 진행 바·플레이어 스코프**: `.pzp-pc` 미사용 레이아웃 대비로 플레이어 스코프를 비디오 기준으로 잡고, PZP **진행 슬라이더**를 볼륨 슬라이더보다 우선하도록 선택했습니다. 조상 `opacity`(합성) 계산이 실패할 때 시크 바가 보이면 마커 표시를 유지하는 보조 판별을 추가했습니다. `cmmUiVisible`용 호스트를 `video.parentElement` 대신 플레이어 루트(`.pzp-pc` 등)로 통일했습니다.
- **VOD 마커·하단바 동기화**: 마커/클러스터 UI 위에서 PZP 하단 크롬이 사라지지 않도록 포인터 nudge 조건을 보강했습니다. 진행 바에 포인터 타깅이 가려지는 경우 좌표/기하 검사를 추가했고, **마커 클러스터 팝이 열린 동안**에는 마우스가 팝 밖으로 나가도 하단바가 유지되도록 nudge를 계속 수행합니다.

## [0.1.2] - 2026-04-10

### 사용자 안내 (제한)

- **라이브 카테고리 자동 기록**은 **일반(기본) 레이아웃**에서만 기대할 수 있습니다. **전체 화면**·**넓은 화면 보기(시어터)** 에서는 치지직 DOM에 카테고리 후보가 잘 붙지 않아 자동 감지·자동 기록이 동작하지 않을 수 있습니다. 방송 **경과 시각** 등은 같은 탭 안 **페이지 DOM·화면에 노출된 텍스트**만 사용합니다(치지직 API 호출 없음).

### Added

- `tabs` 권한: 팝업에서 **현재 활성 탭 URL**로 치지직 VOD 여부를 판별하고, 선택한 세션과 **타임라인 연결**에 사용합니다.
- 팝업 **플레이어 도구 표시** 설정: 메모·타임라인 연결(🔗)을 치지직 플레이어에 보일지 저장(`chzzkPlayerToolsVisibility`). **연결(🔗)은 VOD에서만** 표시합니다.
- 팝업에서 현재 탭이 치지직 `/video/…`일 때, 선택 세션으로 타임라인 연결(`tabs` 우선, 실패 시 `chzzkActivePageContext` 폴백).
- **TXT Import** 세션에 VOD 연결 시: 스트리머·세션 제목 반영 및 저장소 갱신 전 **확인 대화상자**(원본 TXT는 자동 변경 없음).
- 팝업 **라이브 카테고리 자동 감지 ON/OFF** (`chzzkCategoryAutoDetect`, 기본 ON). 끄면 주기 감시·진입 시 자동 기록이 멈추고, 다시 켜면 라이브 탭에서 즉시 재개합니다.

### Changed

- **README**·**개인정보처리방침**: `https://chzzk.naver.com/*` 범위만 사용, **치지직 API·별도 원격 수집 없음**, `tabs`·플레이어 버튼·카테고리 자동 감지 설정 반영.
- 팝업·연결 패널 **스타일** 보강(`popup.css`, `styles.css`).
- VOD 타임라인 마커·팝업 목록에서 카테고리 자동 기록 항목을 **「카테고리」/「카테고리 변경」** 과 이름으로 구분해 표시.

### Fixed

- **라이브 카테고리 오탐**: 카테고리 자동 기록에 채팅 닉네임·채팅 내용이 들어가던 문제.
  - *원인*: `getDefaultCategory()`가 문서 전역의 `a, span, div`를 훑은 뒤, 짧은 텍스트 중 키워드(게임·롤 등)가 맞는 첫 노드를 카테고리로 채택해 채팅 한 줄과 충돌.
  - *조치*: 전역 키워드 스캔 제거. 방송 정보 줄(`video_information_*` 등) 우선, `a[href*='/category/']` 위주. 채팅·댓글 패널 subtree는 후보에서 제외(`isLikelyChatOrCommentSubtree`).
- **라이브 카테고리 시각**: 방송 경과가 아니라 시청·재생 기준처럼 어긋나던 문제.
  - *원인*: 방송 경과 시각을 페이지 전역의 `video_information_count` span 중 첫 시계 문자열로만 판별해 오탐 가능. DOM 실패 시 `video.currentTime` 폴백은 라이브 HLS에서 시청자 기준에 가깝게 동작할 수 있음.
  - *조치*: `video_information_row` / `data` / `status` 안으로 스코프 한정. `스트리밍`·`streaming`·`방송 중` 등 라벨이 있는 span의 시각을 우선 파싱한 뒤, 같은 영역 내 일반 시계 문자열로 재시도.
- **라이브 경과 시각**: 넓은 화면·전체 화면용 **`live_information_player_*`** 등 DOM을 추가로 활용하고, `스트리밍 중` 등 **라벨이 붙은 시계 텍스트**를 우선합니다. 인라인 JSON `openDate`·짧은 `MutationObserver` 프로브·앵커 캐시 재검증을 보강했습니다.
- 인라인 스크립트 등에서 읽은 **`"YYYY-MM-DD HH:mm:ss"` 형식 `openDate`** 파싱 보정.
- **라이브 카테고리·게임 링크**: 위 오탐 방지 조치에 더해 `live_information_player_*` 영역, 인라인 스크립트의 `liveCategoryValue`·`categoryValue` 등 JSON 패턴, `chzzk.naver.com/category/`·`/category/` 셀렉터를 보강했습니다.
- **플레이어 UI**: 타임라인 연결 버튼을 항상 DOM에 두고 **VOD에서만** 표시해 마운트 시점 오인을 줄였습니다.
- **팝업**: 푸터 토글 레이아웃, 확장 팝업에서 높이가 무너지지 않도록 **`vh` 의존 제거**, VOD 연결 패널 상시 표시·비활성 처리, 확인 모달 버튼 문구 정리.
- **`#__NEXT_DATA__` 전용 파싱 분기** 제거, 인라인 스크립트 등 기존 경로로 통합.
